1 GeneSelectR Quarto Website - Deployment Guide

1.1 ✅ Deployment Status: COMPLETED

Live Website: https://jixing475.github.io/GeneSelectR/

Repository: https://github.com/jixing475/GeneSelectR

Deployment Date: 2025-11-04

1.2 What Was Deployed

1.2.1 Main Branch (main)

Contains the R package source code and Quarto documentation source: - R package files (R/, man/, tests/, etc.) - Quarto source files (quarto/*.qmd) - Configuration (quarto/_quarto.yml) - Generated site (quarto/_site/) - tracked for reference

1.2.2 GitHub Pages Branch (gh-pages)

Contains only the compiled website (static HTML): - 13 HTML pages (11 chapters + index + BUILD_SUCCESS) - site_libs/ directory with Bootstrap, Quarto assets - search.json for site search - .nojekyll file (prevents Jekyll processing)

1.3 Deployment Method

Used GitHub Pages with gh-pages branch: 1. Website built locally with quarto render 2. Output copied to separate gh-pages branch 3. Branch pushed to GitHub 4. GitHub Pages automatically serves from gh-pages branch

1.5 Updating the Website

1.5.1 Quick Update (Minor Changes)

  1. Edit .qmd files in /Users/zero/Desktop/GeneSelectR/quarto/

  2. Re-render: cd quarto && quarto render

  3. Deploy:

    cd /tmp && rm -rf geneselectr-deploy
git clone https://github.com/jixing475/GeneSelectR.git geneselectr-deploy
cd geneselectr-deploy
git checkout gh-pages
rm -rf *
cp -r /Users/zero/Desktop/GeneSelectR/quarto/_site/. .
rm -rf _site  # Remove nested _site if exists
touch .nojekyll
git add -A
git commit -m "docs: Update website"
git push origin gh-pages

1.5.2 Full Rebuild

cd /Users/zero/Desktop/GeneSelectR/quarto
quarto render
# Then follow deploy steps above

1.6 GitHub Pages Configuration

  • Source: gh-pages branch, / (root) directory
  • Custom domain: None (using default github.io)
  • HTTPS: Enforced
  • Build type: Legacy (manual deployment, not GitHub Actions)

1.7 Troubleshooting

1.7.1 Website not updating

  1. Check deployment status: https://github.com/jixing475/GeneSelectR/deployments
  2. Hard refresh browser (Cmd+Shift+R on Mac, Ctrl+F5 on Windows)
  3. Wait 1-2 minutes for GitHub CDN to refresh

1.7.2 Styles not loading

Ensure site_libs/ directory is present in gh-pages branch:

git checkout gh-pages
ls site_libs/
# Should show: bootstrap, clipboard, quarto-html, quarto-nav, quarto-search

1.7.3 Broken links

Check that all .qmd files rendered successfully:

cd /Users/zero/Desktop/GeneSelectR/quarto
quarto render 2>&1 | grep -i error

1.8 Manual Push (If Needed)

If automated deployment fails:

cd /tmp/geneselectr-deploy
git push -f origin gh-pages

Note: Force push (-f) is safe here because gh-pages is deployment-only.

1.9 Verification

Check that your website is live:

curl -I https://jixing475.github.io/GeneSelectR/
# Should return: HTTP/2 200

Or open in browser:

open https://jixing475.github.io/GeneSelectR/
# macOS

# Or
gh repo view jixing475/GeneSelectR --web

1.10 Important Notes

  1. Source vs Deployment:

    • main branch = source code
    • gh-pages branch = deployed website

  2. Don’t edit gh-pages directly: Always edit .qmd files, render, then deploy

  3. Caching: Quarto caches results. To force rebuild:

    quarto render --cache-refresh

  4. Assets: The site_libs/ directory contains ~700KB of assets. This is normal for Quarto sites.
